# 小说纠错处理系统 - 预处理阶段

## 📋 项目概述

本项目是一个小说自动纠错系统的预处理模块。第一阶段完成了 **文件读写模块** 和 **文本预处理模块** 的开发与测试。

## 🔧 模块说明

### 1. 文件读写模块 (`modules/file_handler.py`)

**功能：**
- UTF-16编码格式的文本文件读取
- UTF-16编码格式的文本文件写入（保持Unix LF换行符）
- 目录文件扫描与批量读取
- 完善的异常捕获与错误日志

**主要类：**
- `FileHandler`: 文件读写处理类
  - `read_file()`: 读取单个文件
  - `write_file()`: 写入单个文件
  - `read_directory()`: 扫描目录获取匹配文件列表

### 2. 文本预处理模块 (`modules/text_preprocessor.py`)

**功能：**
- 移除非换行符的空白字符（空格、制表符、回车等）
- 将连续多个换行符替换为单个换行符
- 详细的预处理统计信息

**预处理流程：**
```
原始文本
  ↓
[第一步] 移除非换行符的空白字符
  ↓  正则: r'[^\S\n]'
[第二步] 合并连续换行符
  ↓  正则: r'\n+'
处理后的文本
```

**统计信息：**
- 原始文本长度
- 处理后文本长度
- 移除的字符数
- 原始换行符数量
- 处理后换行符数量
- 换行符减少数量

### 3. 预处理主程序 (`main_preprocess.py`)

**功能：**
- 批量处理所有小说文件
- 自动创建输出目录
- 完整的日志记录
- 统计信息保存为JSON格式

**执行流程：**
```
1. 初始化日志系统
2. 扫描 ori_story/ 目录获取所有.txt文件
3. 对每个文件执行：
   - 读取文件（UTF-16编码）
   - 预处理文本
   - 写入文件（UTF-16编码，LF换行符）
   - 记录统计信息
4. 输出处理总结
5. 保存统计信息到JSON
```

## 🚀 使用方法

### 前置条件
- Python 3.7+
- 无第三方库依赖（仅使用标准库）

### 运行预处理

```bash
# 进入项目目录
cd novel_correct

# 运行预处理主程序
python main_preprocess.py
```

### 输出说明

**日志文件** (在 `logs/` 目录)：
- `preprocess_YYYYMMDD_HHMMSS.log`: 详细处理日志
- `preprocess_stats_YYYYMMDD_HHMMSS.json`: JSON格式的统计信息

**处理后的文件** (在 `processed_story/` 目录)：
- 10个预处理后的小说文件
- 文件编码：UTF-16 LE
- 换行符：LF (Unix格式)
