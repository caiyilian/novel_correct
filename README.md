# 📖 Novel Correct (小说自动纠错系统)

## 🌟 项目简介
这是一个基于大模型（Ollama + qwen2.5-coder等）的**高精度、防幻觉**小说文本自动纠错系统。
**核心目标**：将小说中所有人物对话的包裹符号强制统一为成对的日文直角引号「 」，同时智能修正错别字、符号缺失和顺序错乱，**绝不篡改原文剧情与格式**。

## 🛠️ 现阶段核心功能 (Features)
本项目已全面升级为 **5-Stage Pipeline（流水线）架构**，支持自动化递进纠错：
- **Stage 1: 暴力全卷扫描** —— 适合初次格式极其混乱的文本，大面积清扫基础错误。
- **Stage 2: 精准规则定位** —— 基于交替规则进行第一轮精准校验与修复。
- **Stage 3: 异常中括号清理** —— 专杀伪装成合法的边界错误（如误判的 `[` `]` 符号）。
- **Stage 4: 最终规则交替兜底** —— 确保全卷格式绝对合规，消除残留错漏。
- **Stage 5: 超长对话深度校验** —— 倒序深度排查因漏符号而意外合并的隐蔽长对话，并进行精准拆分。

## 🚀 如何开始使用 (小白上手教程)

### 第 0 步：环境准备 (Python & 依赖)
1. 确保你的电脑已安装 **Python 3.7 或以上版本**。
2. 在项目根目录下打开终端，安装所需的 Python 依赖：
   ```bash
   pip install -r requirements.txt
   ```

### 第 1 步：配置大模型环境 (重要！)
本项目依赖本地运行的大语言模型进行推理纠错，你需要安装并运行 Ollama。
1. **安装 Ollama**：请前往 [Ollama 官网 (https://ollama.com/)](https://ollama.com/) 下载并安装。如果你是新手，建议先阅读 [Ollama 官方文档 (https://docs.ollama.com/)](https://docs.ollama.com/)。
2. **下载模型**：在终端中运行 `ollama run qwen2.5-coder:14b` 或 `qwen2.5-coder:7b`（根据你的显存大小选择）。
3. **配置系统**：每个人运行的 Ollama 环境和模型名称都可能不一样。在运行程序前，请务必打开 `src/config/settings.py`，确认以下参数：
   - `OLLAMA_HOST`：指向你的 Ollama 服务地址（例如本地通常是 `http://127.0.0.1:11434`）
   - `FALLBACK_MODELS`：你的兜底模型列表（例如 `['qwen2.5-coder:14b', 'qwen2.5-coder:7b']`），**越聪明、参数量越大的模型请放在列表最前面**。
4. **测试连接 (可选)**：如果你不确定配置是否正确，可以运行自带的测试脚本（注意：运行前需进入 `tests/test_ollama.py` 内部确认里面测试用的 IP 和模型名称与你本地一致）：
   ```bash
   python tests/test_ollama.py
   ```

### 第 2 步：放置你的小说文件
请将你需要纠错的小说文本文件（`.txt` 格式），统一放入以下文件夹中：
👉 `data/ori_story/`

### 第 3 步：运行预处理与纠错流水线
项目采用纯净的 CLI 命令行调度设计。打开终端，在项目根目录下运行以下命令：

```bash
# 查看所有可用参数与功能说明
python main.py -h

# ------------------------------------
# 🌟 推荐用法 1：一键批量处理所有小说
# ------------------------------------
# 第一步：一键批量预处理（将 data/ori_story/ 的小说预处理到 data/processed_story/）
python main.py --preprocess_all

# 第二步：一键批量纠错（自动扫描 data/processed_story/ 下的所有小说，跑满 5 级流水线）
python main.py --batch --pipeline

# ------------------------------------
# 🌟 推荐用法 2：单卷测试与调试
# ------------------------------------
# 指定某一个具体的卷进行流水线处理
python main.py -i "data/processed_story/第1卷.txt" --pipeline

# 开启 Debug 模式：在终端实时查看大模型每一次修改的前后对比（非常适合观察模型表现）
python main.py -i "data/processed_story/第1卷.txt" --pipeline --debug
```

## 📂 目录结构说明

```text
novel_correct/
├── main.py                   # 🌟 唯一用户主入口：配置参数、调度 Pipeline 流水线
├── src/                      # 🧩 核心代码库
│   ├── config/               # ⚙️ settings.py 全局环境配置 (大模型IP, 模型列表)
│   └── modules/              # 核心业务逻辑与 5个 Stage 的独立业务引擎
├── docs/                     # 📚 文档中心 (包含最初设计蓝图与历史版本架构归档)
├── data/                     # 📦 数据源
│   ├── ori_story/            # 原始未经处理的小说文件
│   └── processed_story/      # 预处理后的基础文本 (纠错系统的直接输入源)
├── output/                   # 🚀 最终纠错结果产出目录 (细分为 v1 到 v5 五个阶段的输出)
├── logs/                     # 📝 运行日志存放目录 (按日期和时间自动生成，保持控制台整洁)
└── error/                    # ⚠️ 运行过程中大模型报错、超时等异常的记录目录
```
